## I. Prerequisites:

1.  **Application Setup:**
    *   Ensure the main frontend application (Drive Analyzer) is built and running locally (e.g., via `npm run dev` or `bun dev`).
    *   Ensure the new `webhook-server` is set up:
        *   Navigate to the `webhook-server` directory.
        *   Run `npm install` (or `bun install`) to install dependencies.
        *   Run `npm run build` (or `bun run build`) to compile TypeScript.
        *   Run `npm start` (or `bun start`, or `npm run dev` / `bun run dev` for development mode) to start the webhook server. It should typically run on `http://localhost:3001`.
2.  **Tools:**
    *   A modern web browser (e.g., Chrome, Firefox).
    *   Developer tools in the browser for inspecting console logs and network requests.
    *   A text editor for viewing and creating JSON files.
    *   For webhook testing, a service like [webhook.site](https://webhook.site/) or a similar tool (e.g., ngrok if testing a local webhook server from an external service, though for this plan, webhook.site is sufficient for direct calls from the app).
